# 网络安全工单编辑系统

## 系统概述

本系统是基于原有售后工单系统改造的专业网络安全事件处理平台，专门用于记录、跟踪和处理各类网络安全威胁和事件。系统采用现代化的Web界面设计，提供直观的操作体验和专业的安全分析功能。

## 主要功能特性

### 🔒 安全等级管理
- **四级安全等级**：低风险、中风险、高风险、紧急
- **动态等级显示**：实时更新安全等级指示器
- **智能提醒**：根据等级自动调整处理优先级

### 工单删除功能修复 (2024-11)

#### 问题描述
- **现象**：删除工单时提示"删除工单失败"
- **影响**：无法正常删除包含处理流程的工单
- **紧急程度**：高（影响日常运维）

#### 根因分析
1. **数据库约束**：`ticket_flow` 表存在外键约束 `ticket_id` 引用 `ticket.id`
2. **数据完整性**：当工单有关联的流程记录时，直接删除工单会违反外键约束
3. **错误处理**：原代码未处理外键约束异常，导致删除操作失败

#### 解决方案
**修改文件**：`applications/view/system/ticket.py`

**单工单删除函数** (`delete()`)：
```python
def delete():
    # ... 权限检查代码 ...
    
    try:
        # 首先删除相关的工单流程记录（解决外键约束问题）
        from applications.models.ticket_flow import TicketFlow
        TicketFlow.query.filter_by(ticket_id=ticket.id).delete()
        
        # 然后删除工单本身
        db.session.delete(ticket)
        db.session.commit()
        return success_api(msg="删除工单成功")
    except Exception as e:
        db.session.rollback()
        current_app.logger.error(f"Error deleting ticket {ticket_id}: {e}")
        return fail_api(msg="删除工单失败")
```

**批量删除函数** (`batch_delete()`)：
```python
def batch_delete():
    # ... 前置检查代码 ...
    
    try:
        # 首先删除相关的工单流程记录（解决外键约束问题）
        from applications.models.ticket_flow import TicketFlow
        
        for ticket_id_str in ids:
            # ... 权限检查代码 ...
            
            if 有权限删除:
                # 先删除相关的流程记录
                TicketFlow.query.filter_by(ticket_id=ticket.id).delete()
                # 再删除工单
                db.session.delete(ticket)
                deleted_count += 1
        
        if deleted_count > 0:
            db.session.commit()
        return success_api(msg=f"成功删除 {deleted_count} 条工单")
    except Exception as e:
        db.session.rollback()
        return fail_api(msg="批量删除工单失败")
```
